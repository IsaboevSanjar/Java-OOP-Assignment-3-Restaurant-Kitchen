Restaurant Kitchen

Implement a system to manage a kitchen of a restaurant (using Java Collections). The system must meet the following requirements.

R1: Kitchen

The different meals could be prepared in kitchen. Each meal is characterized by its unique ID, name selling price For instance:

Method addMeal() of class RestaurantKitchen let the administrator to add the description of a new meal. To obtain the price of a meal, the method getPrice() is provided. If a price is requested for a meal id that does not exist, the method returns an error code (e.g., -1.0).

R2: Products and refill

The meals of a kitchen are produced by means of product. Each product is characterized by a unique name and the available amount. For instance:

Product name
 Amount
Meat 30000
Oil 10000
Potatoes 50000
Onion 20000
Rice 25000

Products are recharged through the method refillProduct() of class RestaurantKitchen. If a non-existing name is provided a new product is created, otherwise the existing product is recharged. Further, it is possible to query the available amount of a given product with method availableProduct(). If a non-existing product name is queried, the method returns an error code (e.g., -1.0).

R3: Recipes

The meal consist of products for preparing. Meal recipe is generated based on table below and each recipe is characterized by a unique meal ID, product name and product amount For instance:

 Product name
 Amount

Administrator can add recipe information based on meal ID, product name to be used and product amount. To this aim method addRecipe() is provided.
Given the name of a meal (e.g., "Plov") it is possible to obtain the total number of available products to prepare current meal by means of method availableMeal(). The method counts all products to prepare meal and return boolean value with result.

R4: Selling

The restaurant kitchen sells a product by specifying the meal ID in method sell(). The method returns the boolean value id meal were able to be prepared and sold. If code is invalid (meal id), the meal ran out, an error code is returned (e.g., -1). On normal behavior, the kitchen decrements the quantity of available meal products.
Also implement method preparedMeal() which returns List of all meals which was prepared in Descending order by number of preparation by mean of meal name and number of preparations
